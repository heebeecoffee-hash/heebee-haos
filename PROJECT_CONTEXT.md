# HAOS — Project Context

**Project:** Heebee Asset OS (HAOS)
**Owner:** Gavish Batra (founder)
**Co-owner:** Nitigya
**Company:** Heebee Coffee Pvt. Ltd.

---

## What it is

Single web app to track every owned asset across all Heebee outlets — coffee machines, ACs, grinders, fridges, furniture, crockery — plus the vendors who service them, the utilities each outlet pays, and the repair workflow that approves and tracks every fix.

---

## Stack (locked)

- **Backend:** Google Apps Script (`code.gs`) — pasted manually into script.google.com
- **Database:** Google Sheet — 13 tabs (created automatically by setup function on first run)
- **Frontend:** Single-page PWA (`index.html`) — hosted on GitHub Pages → Vercel
- **File storage:** Google Drive — folder `HAOS_Assets/` with sub-folders per asset
- **Service worker:** `sw.js` — network-first, ~10s propagation
- **Manifest:** `manifest.json` — installable PWA

## Asset code format

`HB-{outlet}-{cat}-{##}` — e.g. `HB-GHB-AC-01`, `HB-SHB-CM-03`, `HB-JLD-GR-02`

Auto-generated by backend. Increments per outlet+category combo.

## Roles

| Role     | Sees values? | Can approve? | Can create users? | Can add assets? |
|----------|--------------|--------------|-------------------|-----------------|
| Owner    | ✅           | ✅           | ✅ (sheet only)   | ✅              |
| Manager  | ❌           | ❌           | ❌                | ✅              |

Future roles (e.g. Barista — QR-scan only) added from `Users` sheet `role` column. Roles drive the UI — backend enforces access on every read/write.

**Users created from backend sheet only** — never from the app UI.

## Sheet tabs (created on first setup run)

1. **Users** — email, name, pin, role, active, created_at
2. **Config** — key/value pairs (drive root folder ID, app version, etc.)
3. **Outlets** — code, name, seats, address, active
4. **Categories** — code, name, default_serviceable, active
5. **Assets** — full asset registry
6. **Stockyard** — assets currently not in active service
7. **Service_Log** — every preventive service performed
8. **Repair_Journal** — every repair, with workflow states & cost
9. **Vendors** — repairmen, electricians, AC techs, AMC partners
10. **Utilities** — electricity, diesel, water per outlet per month
11. **Tasks** — assignments with aging
12. **Approvals** — owner approval queue + history
13. **Audit_Log** — every edit/create/delete with user + timestamp

---

## Repair workflow states

```
Reported → Owner Approved → Vendor Assigned → In Progress → Completed → Verified
```

**Owner approves BEFORE vendor is called.** Each state stamped with date + user.

---

## Auth

- Email + 4-digit PIN (auto-submit on 4th digit)
- Session in `localStorage` as `hb_haos_session`
- 2-hour auto-logout
- 30-second ping for "online now" presence
- Same pattern as Heebee Shift System v3
